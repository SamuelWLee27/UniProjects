When to use Get and Post?

GET vs POST

The GET and POST methods are two of the most common HTTP methods. The main difference between them is GET requests data from the server where the POST method is used to send data for the server to process.

GET method

The HTTP GET method retrieves a specified resource from a server. This means it doesn't need a body because it does not need input data. As GET is only used to request data. When using a browser the GET method is used to download the files of the web page so the client can display them. When using the GET method you can only receive data not change its state. Therefore, it is only used to view a resource not change it. Since this data is stored in the URL that means it can be cached and remain in the browser history. But since it uses the URL the amount of data sent is limited.

When to use the GET method

The GET is used when you want to view a recourse from a server. Normally a web page which includes HTML documents, CSS files, images and videos. The requested data from the server then is sent to the client then displayed.

POST method

The HTTP POST method sends data to the server to be processed. Meaning the request message must have a request body. The request body contains the data to be processed by the server. The POST method is mainly used to create or update a resource. Which means it can be used to add to a particular resource/data. This data is more secure then when using a Get method as it is displayed in the HTTP message body not in the URL. Therefore, it's not stored in logs or browser history. Also, since POST uses the message body there is no limit to the amount of data it can send.

When to use the POST method

The POST method is used when you want to send data to the server. It is mostly used to submit forms or upload files to a server. This is because it is a lot more secure as it is not exposed in the URL bar and a lot more information can be sent as it uses the request body.
