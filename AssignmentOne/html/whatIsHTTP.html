What is HTTP?

HTTP

HTTP is an abbreviation for hypertext transfer protocol. HTTP is used to define how entities dealing with the movement of hypertext documents on the network should communicate with each other.

  • Entities are web browsers and web servers (software applications).
  • HyperText documents are web pages and the resources they use
  • The network is the local network or something broader or worldwide

Basically HTTP is the main method used to transfer data from web pages over a network. Servers store web pages, which are then accessed by the user making a client request by searching something. Without HTTP, the world wide web (www) would not exist. As it is created from server client connections forming a network.

A big issue with HTTP is that data over a HTTP connection is not encrypted, so it runs the risk of third-party interference from attackers trying to steal information. As information over a network using HTTP is not private, sensitive information like credit card data is vulnerable if submitted on a HTTP page.

How it works

A request message describing what the client is trying to retrieve is sent by the browser to the server. Then a response message is sent by the server to the browser describing how it coped with the request, along with the requested resource if available. This all takes place in a manner of milliseconds. So, for example you search www.cms.waikato.ac.nz. The request for this web page is sent to the server. Then the server sends a response back with the result of the request. Then the results are displayed by the client to the user.

How does HTTPS work?

HTTPS stands for hyper transfer protocol secure. Unlike HTTP, HTTPS is encrypted. HTTPS uses SSL (secure sockets layer) which is able to secure a connection and verify that the site is legitimate. SSL creates a secure, encrypted connection between browser and a server, that is able to protect the layer of communication between the two.

HTTPS is important because it adds a layer of protection to prevent third-party attackers from getting important sensitive data. This is extremely important for websites that involve transactions or when your users need to enter their real information.

Different versions of HTTP

  • HTTP v0.9

    This is the first documented version of HTTP (1991). It has only one method, GET (a web page), and server response is always HTML web page

  • HTTPS

    Also called SSL, HTTPS is the encrypted variant of HTTP to guard against hacker attacks. It was invented in 1995, but was not widely used until recently

  • HTTP v1.1

    This is the most commonly used HTTP. It has increased security, support for additional methods (e.g. PUT), and different types of responses (e.g. images)
