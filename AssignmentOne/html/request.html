What is a request?

Basics of a HTTP request

A HTTP request is made by a client, to a named host, which is located on a server. The goal is to access a resource on a server for example a web page.

A HTTP request is composed of three elements:

  • A request line.

  • A request of specific headers.

  • An optional request body.

Request line

The request line consists of three things:

  • The HTTP method (the operation we want to carry out on the resource). The method is a one word command that tells the server what it should do with the resource requested.

  • The path and filename of the resource being requested.

  • The version of HTTP to use.

HTTP request body

Sometimes the request body plus the header request do not always contain all the information needed. This is when a request body comes into play. As request bodies are only appropriate for some request methods. For example, a POST method where the body of the request message is a string representing the data entered by the user. Where as the GET method, which asks the server to send a recource, does not have a request body

Common error codes

There are two status code categories that when recieved in a response message means an error has occured. These are client errors (4xx: bad request syntax, can't be fulfilled) and server errors (5xx: request seende valid but could not fulfill it).
